# Пешеходный переход (Pedestrian Crossing)

Моделирование работы регулируемого пешеходного перехода с вероятностными нарушениями, аварийными ситуациями и выездом аварийной службы.  
Приложение написано на C# с использованием WPF, реализует паттерн MVVM и требования объектно-ориентированного программирования.

## Условие задачи

Смоделировать работу пешеходного перехода.  
Реализовать:
- Классы **Авто** (`Car`), **Пешеход** (`Pedestrian`);
- Интерфейс **аварийная служба** (`IEmergencyService`);
- События: **зеленый свет для пешехода**, **аварийная ситуация** (с некоторой долей вероятности) – выезд машины аварийной службы.

Дополнительные требования:
- Делегаты / события;
- Многопоточность;
- Рефлексия (где необходимо);
- Динамическое движение моделей на форме;
- Возможность добавлять несколько моделей;
- Классы в отдельном модуле;
- Оконное приложение с паттерном MVVM.

## Реализация

### Архитектура решения

- **PedestrianCrossing** – WPF‑приложение (представление и ViewModel).
- **PedestrianCrossing.Models** – библиотека классов (модель, симуляция, интерфейсы).

Проекты связаны ссылкой: WPF ссылается на библиотеку.  
Логика работы полностью вынесена в `PedestrianCrossing.Models` и не зависит от UI.

### События и делегаты

- **`GreenLightChanged`** у `TrafficLight` – оповещает об изменении сигнала светофора. Используется в симуляции и для отображения цвета в интерфейсе.
- **`EmergencyOccurred`** у `CrossingSimulation` – генерируется при фиксации столкновения машины с пешеходом в зоне перехода.
- **`EmergencyVehicleDispatched`** у `EmergencyService` – вызывается при отправке машины аварийной службы.
- Делегат `UIUpdateAction` используется для безопасного обновления UI из фонового потока симуляции.

### Вероятность аварии

Водитель с заданной вероятностью (`IgnoreRedProbability`, по умолчанию 8 %) может проигнорировать красный сигнал светофора. Решение принимается однократно при подъезде к переходу и действует, пока машина не покинет зону перехода. Если в этот момент на дороге находится пешеход, происходит столкновение (авария).

### Многопоточность

Симуляция работает в отдельной задаче (`Task.Run`), что не блокирует интерфейс.  
Все изменения координат и состояний моделей производятся в фоновом потоке, а обновление UI – через `Application.Current.Dispatcher.Invoke`.  
При возникновении аварии симуляция временно останавливает движение обычных машин и пешеходов, пока аварийная машина не прибудет на место.

### Рефлексия

- Используется для поиска всех неабстрактных наследников класса `Vehicle` в сборке моделей.  
- Пользователь может выбрать тип транспортного средства в выпадающем списке, и новый объект создаётся динамически через `Activator.CreateInstance`.
